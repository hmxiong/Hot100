# 算法方向（复杂度 / 数据结构 / 常见套路）

## 高频 Q&A

- Q：如何计算一个函数的时间复杂度和空间复杂度？
  A：先选定输入规模（n/m/k 等），再用“大 O”只保留主导项来描述增长趋势。时间复杂度看“基本操作次数随规模如何增长”，空间复杂度看“额外占用的内存随规模如何增长”（区分输入本身与辅助空间）。

  - 时间复杂度（Time Complexity）计算步骤：
    1. 明确输入规模：例如数组长度 n、矩阵行列 m×n、树节点数 N、边数 E、字符串长度 L 等；
    2. 定义基本操作：常用是比较/赋值/访问/加减乘除/一次 hash 查询等；
    3. 数循环层数：顺序语句相加、循环是迭代次数相乘；只取最高阶；
    4. 处理分支：取最坏/平均/最好（面试默认最坏）；多分支通常取 max；
    5. 处理递归：写出递推式 T(n)，用主定理/递推展开/代入法求解；
    6. 注意摊还：例如动态数组扩容、并查集路径压缩，单次可能慢但均摊更小。

  - 常见时间复杂度速查：
    1. 单层循环：for i in [0..n) → O(n)
    2. 双层嵌套：for i..n for j..n → O(n^2)
    3. 对半/倍增：i*=2 或二分 → O(log n)
    4. 分治：T(n)=2T(n/2)+O(n) → O(n log n)（归并排序）
    5. 递归深度 n：T(n)=T(n-1)+O(1) → O(n)
    6. 图：BFS/DFS → O(V+E)

  - 空间复杂度（Space Complexity）计算步骤：
    1. 区分输入空间与辅助空间：面试常说空间复杂度默认指“辅助空间”（auxiliary space）；
    2. 统计额外数据结构：数组/哈希表/队列/栈/并查集等；
    3. 统计递归栈：递归深度 h，则栈空间 O(h)；
    4. 注意输出是否计入：很多题默认不把“必须返回的输出”算入额外空间，但要说明口径（如返回新数组 vs 原地修改）。

  - 典型例子：
    1. 二分查找：时间 O(log n)，空间 O(1)（迭代）/ O(log n)（递归栈）
    2. 快排：平均时间 O(n log n)，最坏 O(n^2)；空间平均 O(log n)（递归栈），最坏 O(n)
    3. 哈希表去重：时间平均 O(n)，空间 O(n)

- Q：如下递归函数 foo 的时间复杂度是多少？
  A：O(2^n)。

  - 递推式：T(n)=2·T(n-1)+O(1)，T(0)=O(1)
  - 展开：T(n)=2^n·T(0)+O(2^n)=O(2^n)
  - 直观：每层递归分裂成 2 个子问题，递归树共有约 2^n 个叶子/调用

- Q: 讲述一下PPO、DPO以及GRPO算法的发展过程

  PPO、DPO、GRPO这条线可以按一个主线来理解： 怎么把“人类偏好/规则”变成可优化的目标，并且在大模型上稳定、低成本地优化 。它们的演化基本是在权衡三件事：训练稳定性、工程复杂度（尤其是是否需要 critic）、以及对偏好数据形态的适配。

  ## 1) PPO：从通用RL到RLHF的“可控强化学习”
  背景与动机

  - 强化学习里想最大化期望回报 [ o bj ec tO bj ec t ] E [ R ] ，但直接 policy gradient 方差大、容易崩。
  - PPO（Proximal Policy Optimization）提出“别一次更新太猛”，用 clip 约束策略更新幅度，训练更稳。
  在 RLHF 里的形态

  - 先用人工偏好训练一个 Reward Model (RM) 给生成结果打分 [ o bj ec tO bj ec t ] r RM ​ ( x , y ) 。
  - 真正训练时还会加上对参考模型的 KL 约束防跑偏：
    [
    r(x,y)=r_{\text{RM}}(x,y)-\beta,\mathrm{KL}(\pi_\theta(\cdot|x)|\pi_{\text{ref}}(\cdot|x))
    ]
  - PPO 需要 Actor-Critic ：
    - Actor：策略 [ o bj ec tO bj ec t ] π θ ​ 负责生成
    - Critic：价值函数 [ o bj ec tO bj ec t ] V ϕ ​ 负责估计回报，用于优势 [ o bj ec tO bj ec t ] A 降方差
  优缺点

  - 优点：非常稳，理论与工程都成熟。
  - 缺点：需要 RM + Critic + rollout，系统复杂、成本高，训练链条长（RM 误差、critic 误差都可能带来不稳定/偏差）。
  ## 2) DPO：把“偏好优化”从RL改写成一个监督式logistic目标
  动机

  - PPO-RLHF 的主要成本和复杂性来自：rollout + critic + GAE + PPO 多项损失的调参。
  - 人类偏好数据常是 pairwise：同一 prompt 下 chosen/rejected。能不能直接用这种数据训练，而不做显式 RL？
  核心思想

  - DPO（Direct Preference Optimization）把 KL 正则化的 RL 目标推导成一个 等价的偏好分类（logistic）损失 ，不需要显式 reward 训练循环和 critic。
  - 对样本 [ o bj ec tO bj ec t ] ( x , y + , y − ) ：
    [
    \mathcal{L} {\text{DPO}}
    =-\log\sigma\Big(\beta[(\log\pi \theta(y^+|x)-\log\pi_\theta(y^-|x))-(\log\pi_{\text{ref}}(y^+|x)-\log\pi_{\text{ref}}(y^-|x))]\Big)
    ]
  - 直觉：让模型相对参考模型“更偏好 chosen 而不是 rejected”， [ o bj ec tO bj ec t ] β 控制力度。
  优缺点

  - 优点：不需要 critic；训练更像 SFT，工程简单、便宜、稳定性通常更好。
  - 缺点：更依赖偏好对数据质量；对复杂的 credit assignment（长链推理中每步贡献）不如显式RL直观； [ o bj ec tO bj ec t ] β 和长度偏置等问题依然要处理。
  ## 3) GRPO：把“优势估计”从Critic改成“组内相对基线”
  动机

  - DPO 虽然省掉 critic，但它主要吃“成对偏好数据”。如果你有的是 可打分的结果（reward/verifier） ，并且希望像 PPO 一样利用“优势”信号更新策略，同时又不想训练 critic，怎么办？
  - 于是出现了“用同一 prompt 下多条采样的 组内比较 来构造优势”的思路：这就是 GRPO 的核心精神。
  核心思想（概念层面）

  - 对每个 prompt [ o bj ec tO bj ec t ] x ，采样一组回答 [ o bj ec tO bj ec t ] { y i ​ } i = 1 G ​ ，用打分器（RM/规则/验证器）得到 [ o bj ec tO bj ec t ] { r i ​ } 。
  - 用组内统计量当 baseline 来构造相对优势，例如常见形式是：
    [
    A_i = \frac{r_i-\text{mean}(r)}{\text{std}(r)+\epsilon}
    ]
    或 [ o bj ec tO bj ec t ] A i ​ = r i ​ − mean ( r ) （不同实现会有差异）。
  - 然后做类似 PPO 的 ratio 更新（通常也带 KL 到参考模型），但 不需要学习一个 [ o bj ec tO bj ec t ] V ϕ ​ ，因为 baseline 来自同组样本的相对比较。
  直观理解

  - PPO 的 critic 是“学出来的 baseline”；
  - GRPO 的 baseline 是“采样出来的 baseline”（同一 prompt 的多样本相互当参照）。
    这可以在很多场景显著降低系统复杂度，同时保留“优势驱动”的更新形态。
  优缺点

  - 优点：不训练 critic（省复杂度与不稳定源），对“能打分但缺偏好对”的数据形态友好；组内标准化往往让训练更稳。
  - 缺点：需要每个 prompt 多采样（组大小 G 带来算力开销）；优势质量高度依赖打分器/验证器；组内归一化方式、KL 强度等超参依然敏感。
  ## 把演化过程总结成一句话
  - PPO ：通用 RL 稳定化（clip + actor-critic），在 RLHF 中用 RM 提供奖励、critic 提供优势，强但系统重。
  - DPO ：把“KL 正则化的偏好优化”改写成监督式 logistic，去掉 critic/rollout，简单省钱。
  - GRPO ：在不训练 critic 的前提下恢复“优势”更新，用同 prompt 的多采样组内相对基线来做 policy optimization，介于 PPO 的表达力与 DPO 的工程简化之间。

- Q: PreNorm 和 PostNorm
  A: prenorm： x_(t+1) = x_t + F_t(Norm(x_t))
       postnorm： x_(t+1) = Norm(x_t + F_t(x_t))

    分别的优势：prenorm比较训练，但是深度效果并不好：PreNorm比较偏重来自底层的恒等分支。恒等分支更容易训练。
              postnorm难训练，但是深度效果好：PostNorm比较偏重来自高层的线性分支。线性分支更难训练。
